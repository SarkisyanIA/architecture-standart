### <a name="_qmphm5d6rvi3"></a>**Решение**
[Диаграмм контекста](Task4/Context_diagram_task4.puml)
[Диаграмма контейнеров](Task4/Container_diagram_task4.puml)
Выбранны именно эти технологии, т.к у команды есть опыт работы с ними.
Для партнерского кол-центра принято решение передавать XLS-файл с актуальными депозитными ставками по почте, т.к они во внешнем контуре и не предусматривается из взаимодействие по API. Кол-центр банка может узнавать актуальную ставку по депозитам напрямую из АБС.
### <a name="_bjrr7veeh80c"></a>**Альтернативы**
Можно было по другому отправлять актуальные ставки партнерскому кол-центру, например по СМС или другим способом.

**Недостатки, ограничения, риски**
- Email — ненадежный канал для передачи структурированных бизнес-данных. Файл может быть переслан, сохранен на незащищенных устройствах, перехвачен.
- Отсутствие Non-Repudiation: Сложно доказать, что конкретный сотрудник получил или не получил данные.
